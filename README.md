# Open Claw Infrastructure

Infrastructure to run openclaw on AWS Lightsail.

## AWS Infrastructure

An AWS Pulumi project to install, run and operate [openclaw](https://github.com/openclaw/openclaw) on AWS Lightsail.

### What this creates

| Resource | Detail |
|---|---|
| Lightsail instance | `small_3_0` — 2 vCPU, 2 GB RAM, 60 GB SSD, ~$10/month |
| Static IP | Attached to the instance — free while running |
| SSH key pair | Auto-generated by Pulumi — private key exported as secret |
| Firewall | SSH (port 22) only — all other inbound blocked |

**Estimated cost: ~$10/month** (Static IP is free when attached.)

### Why SSH over SSM

AWS SSM Session Manager was evaluated but requires the **advanced-instances tier** for hybrid/on-premises managed instances (which Lightsail registers as), adding ~$5/month. For a single personal instance, SSH with key-based auth is simpler and cheaper.

### Why Ubuntu 22.04 LTS

- **NodeSource support**: NodeSource publishes official APT packages for Node.js 24 with first-class Ubuntu support — the simplest way to get the Node version openclaw requires.
- **LTS stability**: Supported until April 2027. No surprise EOL during a long-running personal project.
- **Lightsail availability**: Ubuntu 22.04 is a first-class Lightsail blueprint in all regions including eu-central-1.
- **Community**: The vast majority of Linux server documentation targets Ubuntu, making troubleshooting straightforward.

### Why a static IP

openclaw connects outbound to 20+ messaging platforms (Telegram, WhatsApp, Slack, etc.). A static IP ensures the instance always presents the same address to those services — useful if any of them rate-limit or flag rotating IPs.

### Network diagram

```
┌─ Your Machine ─────────────────────────────────────────────┐
│                                                             │
│  Terminal ─── SSH (TCP/22) ──────────────────────────────────────┐
│  Browser ─── SSH tunnel (localhost:18789 → :18789) ─────────┐   │
│                                                             │   │
└─────────────────────────────────────────────────────────────┘   │
                                                              │   │
                                                              ▼   ▼
┌─ Lightsail (eu-central-1) ──────────────────────────────────────────┐
│  Firewall: inbound TCP/22 only, outbound all                       │
│                                                                     │
│  ┌─ Gateway (127.0.0.1:18789) ───────────────────────────────────┐ │
│  │  Dashboard UI · Pi Agent · Session store (~/.openclaw/)       │ │
│  │  Telegram adapter (grammY) ──── outbound long polling ────────────┐
│  └───────────────────────────────────────────────────────────────┘ │ │
└───────────────────────────────────────────────────────────────────┘ │
                                                                      │
                         OUTBOUND ONLY                                │
                         ─────────────────────────────────────────────┤
                                                                      │
           ┌──────────────────┐  ┌─────────────────┐                  │
           │ Telegram Bot API │  │ Anthropic API    │                  │
           │ api.telegram.org │  │ api.anthropic.com│ ◄────────────────┘
           │ HTTPS/443        │  │ HTTPS/443        │
           └──────────────────┘  └─────────────────┘
```

| Connection | Protocol | Port | Direction | Notes |
|---|---|---|---|---|
| You → Lightsail | SSH | TCP/22 | Inbound | Only open inbound port |
| Dashboard | HTTP over SSH | 18789 (local) | Tunnel | Not directly exposed |
| Gateway → Telegram | HTTPS | 443 | Outbound | Long polling (getUpdates) |
| Gateway → Anthropic | HTTPS | 443 | Outbound | AI model API |
| Gateway ↔ itself | WebSocket | 18789 | Localhost | Never touches the network |

## Run this Pulumi Project

while being in project root dir

login to target account:
```
AWS_PROFILE=knip-builds
aws sso login --profile $AWS_PROFILE
```
(if not found type `aws sso configure` and fill in the details. More details about how to set up sso here [this blog post](https://blog.knip-builds.de/posts/aws-sso/))

login to pulumi backend:
```
PULUMI_BACKEND=openclaw-pulumi-backend
pulumi login s3://$PULUMI_BACKEND
```

Export the passphrase for the stack
```
export PULUMI_CONFIG_PASSPHRASE=xxxx
```

```
pulumi stack select prod
```

```
pulumi preview
```

```
pulumi up
```

## Initial Setup
_(in case of disaster or cloud account migration)_

```
PULUMI_BACKEND=openclaw-pulumi-backend
```

```
aws s3 mb s3://$PULUMI_BACKEND
```

```
pulumi login s3://$PULUMI_BACKEND
```

```
pulumi new typescript --name openclaw
```

Define a stack with name
```
pulumi stack init prod
```

Choose a passphrase and export it afterwards. You will get prompted again:
```
export PULUMI_CONFIG_PASSPHRASE=<thePassphrase>
```

## Connecting to the instance

```bash
# Save private key (first time only)
pulumi stack output privateKey --show-secrets > openclaw.pem
chmod 600 openclaw.pem

# SSH in
ssh -i openclaw.pem ubuntu@$(pulumi stack output publicIp)
```

---

## After first connect

The bootstrap script (`scripts/bootstrap.sh`) installs Node.js 24 and openclaw globally. Onboarding is interactive:

```bash
openclaw onboard --install-daemon
```

Follow the prompts to configure your AI model, channels, and workspace.
